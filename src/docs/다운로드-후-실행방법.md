# BLUECARE 프로젝트 다운로드 후 실행 방법

## 📋 필수 사항

이 프로젝트를 다운로드하여 로컬 환경(Cursor, VS Code 등)에서 실행하려면 다음이 필요합니다:

1. **Node.js** (v18 이상 권장)
2. **npm** 또는 **yarn** 패키지 매니저

## 🚀 실행 단계

### 1단계: 프로젝트 다운로드
Figma Make에서 프로젝트를 다운로드합니다.

### 2단계: 터미널에서 프로젝트 폴더로 이동
```bash
cd bluecare
```

### 3단계: 의존성 패키지 설치
```bash
npm install
```

또는 yarn 사용 시:
```bash
yarn install
```

### 4단계: 개발 서버 실행
```bash
npm run dev
```

또는 yarn 사용 시:
```bash
yarn dev
```

브라우저에서 `http://localhost:3000`으로 접속하면 Figma Make에서 보던 것과 동일한 디자인이 표시됩니다.

## 🏗️ 프로덕션 빌드

배포용 빌드를 생성하려면:

```bash
npm run build
```

빌드된 파일은 `dist` 폴더에 생성됩니다.

## 🔧 주요 설정 파일

다음 설정 파일들이 포함되어 있어 Tailwind CSS가 정상적으로 작동합니다:

- **tailwind.config.js** - Tailwind CSS 설정
- **postcss.config.js** - PostCSS 설정  
- **vite.config.ts** - Vite 번들러 설정
- **styles/globals.css** - 전역 스타일 및 Tailwind 임포트

## ⚠️ 중요 사항

### Supabase 환경 변수 설정

프로젝트에서 Supabase를 사용하므로, 로컬 환경에서 실행하려면 `.env` 파일을 생성하고 다음 환경 변수를 설정해야 합니다:

```env
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
VITE_SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key
```

또는 `utils/supabase/info.tsx` 파일에서 직접 값을 설정할 수 있습니다.

## 💡 문제 해결

### 스타일이 적용되지 않는 경우

1. `node_modules` 폴더를 삭제하고 재설치:
   ```bash
   rm -rf node_modules package-lock.json
   npm install
   ```

2. 개발 서버 재시작:
   ```bash
   npm run dev
   ```

### 포트 충돌

기본 포트(3000)가 사용 중이면 `vite.config.ts`에서 포트 변경:
```typescript
server: {
  port: 3001, // 원하는 포트 번호
},
```
