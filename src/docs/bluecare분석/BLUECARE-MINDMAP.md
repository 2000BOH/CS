## 📌 1. 구분 (Category) - 민원 유형 분류

### 🎯 정의
- **역할**: 민원의 "종류"를 나타냄
- **입력 시점**: 민원 등록 시 **필수 선택**
- **데이터베이스**: `complaints.구분` 컬럼에 저장

### 🌳 구분 마인드맵

```
                        구분 (민원 유형)
                              │
        ┌─────────┬───────────┼───────────┬─────────┐
        │         │           │           │         │
   민원(CS)   민원(영선)    입실       퇴실      청소
        │         │           │           │         │
        ↓         ↓           ↓           ↓         ↓
    고객 민원   시설 수리   입실 준비   퇴실 처리  객실 청소
    서비스      설비 관리   점검        정리       정비
```

### 📝 상세 설명

#### 🔹 민원(CS)
- **의미**: Customer Service 관련 민원
- **예시**: 
  - 고객 불만 사항
  - 서비스 요청
  - 프론트 데스크 업무
- **색상**: 🟥 빨간색 계열
- **담당**: CS팀, 프론트 데스크

#### 🔹 민원(영선)
- **의미**: 시설 관리 및 수리 관련 민원
- **예시**:
  - 전기/수도 고장
  - 에어컨 수리
  - 설비 점검
- **색상**: 🟥 빨간색 계열
- **담당**: 영선팀 (07번 계정)

#### 🔹 입실
- **의미**: 새로운 고객 입실 준비
- **예시**:
  - 입실 전 객실 점검
  - 비품 준비
  - 청소 확인
- **색상**: 🟦 파란색 계열
- **담당**: 키핑팀, 프론트

#### 🔹 퇴실
- **의미**: 고객 퇴실 및 정리
- **예시**:
  - 퇴실 점검
  - 객실 상태 확인
  - 손상 확인
- **색상**: 🟨 노란색 계열
- **담당**: 관리사무소 (08번 계정)

#### 🔹 청소
- **의미**: 객실 청소 및 정비
- **예시**:
  - 일반 청소
  - 특별 청소
  - 린넨 교체
- **색상**: 🟩 초록색 계열
- **담당**: 키핑팀 (10번 계정)

## 📌 2. 처리상태 (Status) - 민원 처리 단계

### 🎯 정의
- **역할**: 민원의 "처리 진행 상황"을 나타냄
- **변경 시점**: 민원 진행 중 **수시로 변경**
- **데이터베이스**: `complaints.상태` 컬럼에 저장

### 🌳 처리상태 마인드맵 (6단계 워크플로우)

```
                        처리상태 (워크플로우)
                              │
                         ① 접수
                              │
                              ↓
                    ┌─────────┴─────────┐
                    │                   │
                ② 영선팀           ③ 진행중
                    │                   │
                    │                   │
                    └─────────┬─────────┘
                              ↓
                    ┌─────────┴─────────┐
                    │                   │
                ④ 부서이관          ⑤ 외부업체
                    │                   │
                    │                   │
                    └─────────┬─────────┘
                              ↓
                         ⑥ 완료
```

### 📝 상세 설명

#### ① 접수
- **의미**: 민원이 최초 등록된 상태
- **색상**: 🔵 파란색 (bg-blue-500)
- **조건**: 
  - 민원 등록 시 기본값
  - 아직 누구도 착수하지 않음
- **필터**: "미연락"에 포함

#### ② 영선팀
- **의미**: 영선팀으로 배정된 상태
- **색상**: 🟢 청록색 (bg-teal-500)
- **조건**:
  - 시설 수리/점검 필요
  - 영선팀이 담당
- **다음 단계**: 진행중 또는 완료

#### ③ 진행중
- **의미**: 현재 처리 진행 중
- **색상**: 🟠 주황색 (bg-orange-500)
- **조건**:
  - 작업 착수
  - 처리 중
- **필터**: "미조치"에 포함

#### ④ 부서이관
- **의미**: 다른 부서로 이관
- **색상**: 🟣 보라색 (bg-purple-500)
- **조건**:
  - 해당 부서에서 처리 불가
  - 타 부서 협조 필요
- **필터**: "미조치"에 포함

#### ⑤ 외부업체
- **의미**: 외부 업체에 위탁
- **색상**: 🔷 남색 (bg-indigo-500)
- **조건**:
  - 전문 업체 필요
  - 외주 처리
- **필터**: "미조치"에 포함

#### ⑥ 완료
- **의미**: 민원 처리 완료
- **색상**: 🟢 초록색 (bg-green-500)
- **조건**:
  - 모든 조치 완료
  - 완료일시 자동 기록
- **필터**: "오늘 조치", "이번주 조치"에 포함

## 📌 3. 선택사항 (Category Filter) - 필터링 용어

### 🎯 정의
- **역할**: 전체조회 페이지에서 민원을 **필���링**하는 용어
- **실체**: **"구분"과 동일한 개념**
- **위치**: StatsSidebar 컴포넌트의 두 번째 섹션

### ⚠️ 중요한 발견

```
┌─────────────────────────────────────────┐
│  "선택사항" = "구분" (중복된 개념!)     │
│                                          │
│  입력 시: "구분"이라고 부름             │
│  필터 시: "선택사항"이라고 부름         │
│                                          │
│  → 같은 데이터, 다른 이름!              │
└─────────────────────────────────────────┘
```

### 🌳 선택사항 마인드맵

```
                    선택사항 (필터링)
                          │
            ┌─────────────┼─────────────┐
            │             │             │
        처리상태      구분(선택사항)   날짜필터
            │             │             │
            ↓             ↓             ↓
      전체/접수/      전체/민원(영선)/  전체/오늘 연락/
      영선팀/진행중/  민원(기타)/      오늘 조치/
      부서이관/       입실/퇴실/청소   이번주 연락/
      외부업체/완료                    이번주 조치/
                                       미연락/미조치
```

### 📝 상세 설명

#### 🔹 선택사항 버튼 목록
- **전체**: 모든 구분 표시
- **민원(영선)**: 영선 관련만 필터링
- **민원(기타)**: CS 관련만 필터링 (⚠️ 주의: 입력 시는 "민원(CS)")
- **입실**: 입실 관련만 필터링
- **퇴실**: 퇴실 관련만 필터링
- **청소**: 청소 관련만 필터링

## 🎯 핵심 요약

### 1️⃣ 구분 (Category)
- **목적**: 민원 유형 분류
- **값**: 민원(CS), 민원(영선), 입실, 퇴실, 청소
- **입력**: 등록 시 선택 (필수)
- **변경**: 거의 변경하지 않음

### 2️⃣ 처리상태 (Status)
- **목적**: 처리 진행 단계
- **값**: 접수 → 영선팀 → 진행중 → 부서이관 → 외부업체 → 완료
- **입력**: 등록 시 기본값 (접수)
- **변경**: 처리 중 수시 변경

### 3️⃣ 선택사항 (Filter Label)
- **목적**: UI에서 필터링 시 표시 용어
- **실체**: **구분과 동일**
- **문제**: 용어 중복으로 혼란 야기
- **해결**: "구분"으로 통일 권장
